## Terms & Acronyms

| Term | Meaning |
|------|---------|
| **MLLM** | Multimodal Large Language Model (e.g., BLIP-2) |
| **Triple** | (subject, relation, object) — structured unit extracted from a caption |
| **R-POPE** | Novel metric: extends POPE (object hallucination) to relational Yes/No questions |
| **R-CHAIR_s** | % captions with ≥1 relational hallucination (caption-level) |
| **R-CHAIR_i** | % of total triples that are hallucinated (triple-level) |
| **R-Bench** | Benchmark dataset (Wu et al., ICML 2024) — 11,651 relational Q&A pairs on nocaps images |
| **BLIP-2** | Vision-language model used for captioning + VQA verification (blip2-flan-t5-xl) |
| **OWL-ViT** | google/owlvit-base-patch32 — open-vocab object detection for spatial verification |
| **Mistral-7B** | LLM used for minimal correction step, accessed via Together.ai API |
| **Together.ai** | API provider for Mistral-7B (free tier: 5M tokens/month) |
| **Woodpecker** | Related work — corrects object hallucinations but ignores relational structure |
| **POPE** | Prior benchmark for object hallucination (binary Yes/No); R-POPE extends this |
| **Edit Rate** | Levenshtein(before, after) / len(before) — measures how much correction changed caption |
| **BLEU-4** | Fluency metric — corrected caption vs. reference caption |
| **nocaps** | Image dataset used by R-Bench for evaluation |
| **VQA** | Visual Question Answering — BLIP-2 asked "Is [subject] [relation] [object]?" → Yes/No |
| **RelCheck_Master.ipynb** | Master Colab notebook — runs ALL experiments end-to-end |
| **self-refinement** | Baseline 2: prompt BLIP-2 to re-check its own description |

## RelCheck Pipeline (3 Stages)

1. **Triple Extractor** — spaCy dependency parsing → (subject, relation, object) triples from caption
2. **Relation Verifier** — type-aware: spatial (OWL-ViT + geometry) vs. action/attribute (BLIP-2 VQA)
3. **Minimal Corrector** — Mistral-7B edits only hallucinated spans; self-consistency guard
